# חדר בריחה הפוך — מפצחים את לוויין הריגול

חדר בריחה מדעי בנושא הנדסה הפוכה של לוויין, לתלמידי כיתה ח׳ העובדים בקבוצות של 3–4 במשך 60 דקות.

בשלב זה בנויה **מעטפת המשחק בלבד** — שש החידות עצמן יתווספו בשלב הבא.

## הפעלה

```bash
npm install      # התקנת תלויות (פעם אחת)
npm run dev      # הרצה מקומית לפיתוח — נפתח בכתובת שמודפסת למסך
npm run build    # בניית גרסה לפריסה (נוצרת תיקיית dist)
npm run preview  # תצוגה מקדימה של גרסת ה-build
```

אין צורך בשרת, מסד נתונים או התחברות — ההתקדמות נשמרת ב-localStorage של הדפדפן.

## מה כלול במעטפת

- **מסך פתיחה** — שם קבוצה, תדריך משימה וכפתור "התחל משימה".
- **מסך משחק** — פנורמת 360° של חדר הבקרה (Blockade Labs) הניתנת לגרירה, עם מחוון הסבר.
- **סרגל עליון** — שם הקבוצה, טיימר 60 דקות, התקדמות (0 מתוך 6), השתקת צלילים, איפוס עם אישור.
- **סרגל תחנות** — שש תחנות; רק הראשונה פעילה והשאר נעולות עם סמל מנעול.
- **שמירת התקדמות** — מצב המשחק (כולל הטיימר) נשמר ב-localStorage ושורד רענון דף.

## מבנה הקוד

```
src/
├── components/   # רכיבי מסך: StartScreen, GameScreen, TopBar, StationsBar, PanoramaViewer, Modal
├── puzzles/      # רכיבי חידות — כרגע PuzzlePlaceholder + רישום PUZZLE_COMPONENTS
├── data/         # הגדרות תחנות, תדריך, משך משימה, כתובת פנורמה
├── hooks/        # useGameState (מצב מרכזי + localStorage), useTimer (ספירה לאחור)
├── types/        # טיפוסי TypeScript: GameState, Station, PuzzleProps ועוד
└── styles/       # global.css — ערכת עיצוב של חדר בקרה מדעי
```

## הוספת חידה חדשה

1. יוצרים רכיב חדש ב-`src/puzzles/` שמקבל `PuzzleProps` (מוגדר ב-`src/types/game.ts`).
2. רושמים אותו ב-`PUZZLE_COMPONENTS` שב-`src/puzzles/index.ts` במקום ה-placeholder.
3. הרכיב מקבל `onSolve`, `onUseHint` ו-`onClose` — קריאה ל-`onSolve` פותחת אוטומטית את התחנה הבאה ומעדכנת את ההתקדמות.
